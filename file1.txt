⚠️  PERHATIAN: Wajib Hapus Docker Lama Sebelum Install Ulang: Jika Docker sudah pernah diinstall, HARUS dihapus dulu. Jangan skip atau akan terjadi konflik package yang menyebabkan error saat build.

sudo docker stop $(sudo docker ps -aq)

sudo docker volume rm $(sudo docker volume ls -q)

sudo apt install git -y

1.5 Install Docker
Install dependencies:
sudo apt install ca-certificates curl gnupg lsb-release -y
Tambah GPG key Docker:
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
Tambah repository Docker:
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

⚠️  PERHATIAN: Perintah Echo Harus Satu Baris: Perintah echo di atas harus dijalankan dalam SATU BARIS penuh. Jika terpotong, repo akan salah format dan muncul error 404. Jika error, hapus dulu: sudo rm /etc/apt/sources.list.d/docker.list lalu ulangi.

Update dan install Docker:
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin -y
docker --version && docker compose version
 
BAB 2 — Clone Repository & Konfigurasi Project
2.1 Clone Repository dari GitHub
git clone https://github.com/ershnir/repo-ujikombengkel.git
cd repo-ujikombengkel
2.2 Edit File .env
nano .env
Ubah konfigurasi database berikut:
DB_HOST=db
DB_DATABASE=otoklik
DB_USERNAME=otoklik
DB_PASSWORD=secret

⚠️  PERHATIAN: DB_HOST Harus 'db' Bukan 'localhost': Di Docker, service database diakses via nama service-nya yaitu 'db'. Jika masih 'localhost' atau '127.0.0.1', Laravel tidak bisa konek ke MySQL.
2.3 Buat Dockerfile
nano Dockerfile
FROM php:8.4-fpm

RUN apt-get update && apt-get install -y git curl libpng-dev libonig-dev libxml2-dev libzip-dev zip unzip nodejs npm \
    && docker-php-ext-install pdo_mysql mbstring exif pcntl bcmath gd zip \
    && apt-get clean && rm -rf /var/lib/apt/lists/*

COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

WORKDIR /var/www

COPY . .

RUN composer install --optimize-autoloader --no-dev

RUN npm install && npm run build

RUN chown -R www-data:www-data /var/www \
    && chmod -R 755 /var/www/storage \
    && chmod -R 755 /var/www/bootstrap/cache

EXPOSE 9000

CMD ["php-fpm"]

⚠️  PERHATIAN: Gunakan PHP 8.4, Bukan PHP 8.2: Package Laravel di repo ini membutuhkan PHP >= 8.4. Jika menggunakan FROM php:8.2-fpm, build akan GAGAL.
2.4 Buat docker-compose.yml
File ini tidak menggunakan port 443 atau SSL — semua traffic berjalan di port 80 (HTTP).


nano docker-compose.yml
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    restart: unless-stopped
    working_dir: /var/www
    volumes:
      - ./:/var/www
      - ./storage:/var/www/storage
    networks:
      - network-ujikom
    depends_on:
      - db
    scale: 3
    # healthcheck:                          # Aktifkan jika dibutuhkan
    #   test: ["CMD", "curl", "-f", "http://localhost"]
    #   interval: 60s
    #   timeout: 10s
    #   retries: 3

  nginx:
    image: nginx:alpine
    container_name: ujikom_nginx
    restart: unless-stopped
    ports:
      - '80:80'
    volumes:
      - ./:/var/www
      - ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf
    networks:
      - network-ujikom
    depends_on:
      - app

  db:
    image: mysql:8.0
    container_name: ujikom_db
    restart: unless-stopped
    environment:
      MYSQL_DATABASE: otoklik
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_PASSWORD: secret
      MYSQL_USER: otoklik
    volumes:
      - volume-ujikom:/var/lib/mysql
    ports:
      - '3306:3306'
    networks:
      - network-ujikom

networks:
  network-ujikom:
    driver: bridge

volumes:
  volume-ujikom:
    driver: local

ℹ️  INFO: Healthcheck (Dinonaktifkan): Healthcheck pada service app sudah disiapkan tapi dinonaktifkan (comment). Jika ingin mengaktifkan, hapus tanda # di depan baris healthcheck. Healthcheck akan mengecek aplikasi setiap 60 detik.


2.5 Buat Konfigurasi Nginx
Nginx dikonfigurasi hanya untuk HTTP (port 80). Tidak ada redirect ke HTTPS.
mkdir -p docker/nginx
nano docker/nginx/default.conf
upstream otoklik_app {
    server repo-ujikombengkel-app-1:9000;
    server repo-ujikombengkel-app-2:9000;
    server repo-ujikombengkel-app-3:9000;
}

server {
    listen 80;
    index index.php index.html;
    server_name ujikom-putri.net;
    root /var/www/public;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass otoklik_app;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.ht { deny all; }
}

BAB 5 — Playbook Ansible Versi 1 (All-in-One)
Semua service (DNS + DHCP) dikonfigurasi dalam satu file playbook.
5.1 Buat Playbook
mkdir -p /etc/ansible/playbooks
nano /etc/ansible/playbooks/setup-vm2.yml
- name: Setup VM2 - DNS dan DHCP Server
  hosts: vm2
  become: yes
  tasks:

    # ─── UPDATE SYSTEM ───
    - name: Update apt cache
      apt:
        update_cache: yes

    # ─── INSTALL PACKAGE ───
    - name: Install BIND9 dan DHCP
      apt:
        name:
          - bind9
          - bind9utils
          - bind9-doc
          - isc-dhcp-server
        state: present

    # ─── BIND9 DNS ───
    - name: Configure named.conf.options
      copy:
        dest: /etc/bind/named.conf.options
        content: |
          options {
              directory "/var/cache/bind";
              recursion yes;
              allow-query { any; };
              forwarders { 8.8.8.8; 8.8.4.4; };
              dnssec-validation no;
              listen-on { any; };
          };

    - name: Configure named.conf.local
      copy:
        dest: /etc/bind/named.conf.local
        content: |
          zone "ujikom-putri.net" {
              type master;
              file "/etc/bind/db.ujikom-putri.net";
          };

    - name: Create DNS zone file
      copy:
        dest: /etc/bind/db.ujikom-putri.net
        content: |
          $TTL 604800
          @  IN  SOA ujikom-putri.net. admin.ujikom-putri.net. (
                   2 604800 86400 2419200 604800 )
          @      IN  NS   ujikom-putri.net.
          @      IN  A    192.168.1.10
          www    IN  A    192.168.1.10

    - name: Restart BIND9
      service:
        name: bind9
        state: restarted
        enabled: yes

    # ─── DHCP ───
    - name: Configure DHCP Server
      copy:
        dest: /etc/dhcp/dhcpd.conf
        content: |
          default-lease-time 600;
          max-lease-time 7200;
          authoritative;
          subnet 192.168.1.0 netmask 255.255.255.0 {
              range 192.168.1.100 192.168.1.200;
              option routers 192.168.1.1;
              option domain-name-servers 192.168.1.11;
              option domain-name "ujikom-putri.net";
          }

    - name: Set DHCP interface
      lineinfile:
        path: /etc/default/isc-dhcp-server
        regexp: '^INTERFACESv4='
        line: 'INTERFACESv4="enp0s8"'

    - name: Restart DHCP Server
      service:
        name: isc-dhcp-server
        state: restarted
        enabled: yes
        
5.2 Jalankan Playbook
ansible-playbook /etc/ansible/playbooks/setup-vm2.yml

💡  TIPS: Hasil yang Diharapkan: PLAY RECAP harus menampilkan: failed=0 dan unreachable=0

ℹ️  INFO: Mengapa Docker di VM 2?: Jika VM 2 juga akan menjalankan container di masa mendatang (misalnya scaling), Docker perlu diinstall. Playbook ini mengotomasi proses instalasi Docker Engine lengkap di VM 2.
 
Membuat Playbook docker.yml
Langkah 1 — Buka file baru
nano /etc/ansible/playbooks/docker.yml
Langkah 2 — Isi playbook
Salin seluruh isi berikut ke dalam file:

---
- name: Install Docker di VM 2 (ujikom-managed)
  hosts: vm2
  become: yes
  tasks:

    # ─── UPDATE SYSTEM ───────────────────────────────────────
    - name: Update apt cache
      apt:
        update_cache: yes

    # ─── HAPUS DOCKER LAMA (jika ada) ────────────────────────
    - name: Hapus Docker versi lama
      apt:
        name:
          - docker
          - docker-engine
          - docker.io
          - containerd
          - runc
          - docker-compose
          - docker-compose-plugin
        state: absent
        purge: yes
      ignore_errors: yes

    # ─── INSTALL DEPENDENCIES ────────────────────────────────
    - name: Install dependencies Docker
      apt:
        name:
          - ca-certificates
          - curl
          - gnupg
          - lsb-release
        state: present

    # ─── GPG KEY ─────────────────────────────────────────────
    - name: Buat direktori keyrings
      file:
        path: /etc/apt/keyrings
        state: directory
        mode: '0755'

    - name: Download GPG key Docker
      shell: |
        curl -fsSL https://download.docker.com/linux/ubuntu/gpg |
        gpg --dearmor -o /etc/apt/keyrings/docker.gpg
        chmod a+r /etc/apt/keyrings/docker.gpg
      args:
        creates: /etc/apt/keyrings/docker.gpg

    # ─── REPOSITORY DOCKER ───────────────────────────────────
    - name: Tambah repository Docker
      shell: |
        echo "deb [arch=$(dpkg --print-architecture)
        signed-by=/etc/apt/keyrings/docker.gpg]
        https://download.docker.com/linux/ubuntu
        $(lsb_release -cs) stable" |
        tee /etc/apt/sources.list.d/docker.list > /dev/null
      args:
        creates: /etc/apt/sources.list.d/docker.list

    # ─── INSTALL DOCKER ENGINE ───────────────────────────────
    - name: Update apt setelah tambah repo Docker
      apt:
        update_cache: yes

    - name: Install Docker Engine
      apt:
        name:
          - docker-ce
          - docker-ce-cli
          - containerd.io
          - docker-buildx-plugin
          - docker-compose-plugin
        state: present

    # ─── ENABLE & START DOCKER ───────────────────────────────
    - name: Aktifkan dan jalankan Docker
      systemd:
        name: docker
        state: started
        enabled: yes

    # ─── VERIFIKASI ──────────────────────────────────────────
    - name: Cek versi Docker
      command: docker --version
      register: docker_version

    - name: Tampilkan versi Docker
      debug:
        msg: "{{ docker_version.stdout }}"
 
Cara Menjalankan Playbook
Jalankan docker.yml saja:

Jalankan sekaligus dengan DNS dan DHCP:

Test dulu sebelum jalankan (dry run):
